How It Works

From installer to Intune in one run

1

Place installer & metadata file

Drop the vendor installer (EXE, MSI, or ZIP) into a release folder and add a small metadata file that describes install commands, detection rules, and UAT group assignment.

2

Run the packaging job

Start a new run from the dashboard. The system reads metadata, invokes the Microsoft Win32 Content Prep Tool, and produces the .intunewin artifact alongside logs.

3

Automatic Intune configuration

The app is created in Intune with the correct install/uninstall commands, registry-based detection rules, dependencies, supersedence, and UAT group assignment — all from the metadata file.

Features

What the automation handles for you

Standardized Packaging

Produces .intunewin artifacts consistently from EXE, MSI, or ZIP installers using the official Win32 Content Prep Tool.

Metadata-Driven Config

One small metadata file per release captures install commands, detection rules, dependencies, supersedence, and UAT group — no manual Intune data entry.

Automated Intune Setup

Creates the Win32 app in Intune, uploads the artifact, configures detection rules, sets dependencies and supersedence, and assigns to UAT — all in one run.
